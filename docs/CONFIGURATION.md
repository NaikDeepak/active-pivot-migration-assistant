# Configuration Reference

All settings live under the `migration:` prefix in `application.yml` and are bound to
`MigrationProperties`. Override any value on the command line:

```bash
java -jar migration-assistant.jar --migration.project-root=/path/to/project
```

Or via environment variable (Spring Boot kebab‚Üíupper-snake convention):

```bash
MIGRATION_PROJECT_ROOT=/path/to/project java -jar migration-assistant.jar
```

---

## Core Paths

| Property | Default | Description |
|----------|---------|-------------|
| `migration.project-root` | `${user.dir}` (or `$MIGRATION_PROJECT_ROOT`) | Absolute path to the Java project under migration. Must contain a `pom.xml`. |
| `migration.knowledge-dir` | `~/.migration-assistant/knowledge` | Directory of `.md` / `.txt` migration guide files. Loaded lazily on first use; use `reload` if you add files mid-session. |
| `migration.jar-index-dir` | `~/.migration-assistant/jar-index` | Where `ReflectionJarIndexer` writes one `<jarname>.json` per indexed JAR. |
| `migration.fixes-dir` | `~/.migration-assistant/fixes` | Persisted `FixRecord` JSON files. Grows over time; safe to delete to reset history. |
| `migration.jars-dir` | `~/.migration-assistant/jars` | Drop ActivePivot JAR files here, then run `scan` to index them. |

---

## Maven Settings

| Property | Default | Description |
|----------|---------|-------------|
| `migration.maven-executable` | `mvn` | Maven binary. Override if Maven is not on `PATH` (e.g., `/opt/maven/bin/mvn`). |
| `migration.maven-pre-goals` | `[clean]` | Goals prepended before `compile`. `clean` ensures a fresh build. Remove it for faster incremental compiles on large projects. |

---

## Source Loading

| Property | Default | Description |
|----------|---------|-------------|
| `migration.source-surrounding-lines` | `100` | Lines of context around an error included in the AI prompt. Increase for complex method dependencies; decrease to save tokens. |

---

## AI Settings (`migration.ai.*`)

| Property | Default | Description |
|----------|---------|-------------|
| `migration.ai.provider` | `MOCK` | `MOCK` uses `MockAiProvider` (no API calls, for testing). `CLAUDE` uses the real Anthropic API. |
| `migration.ai.api-key` | _(empty)_ | Anthropic API key. Required when `provider=CLAUDE`. Set via `$ANTHROPIC_API_KEY` to avoid storing in config files. |
| `migration.ai.model` | `claude-sonnet-4-6` | Claude model ID. Change to `claude-opus-4-8` for maximum accuracy on complex APIs. |
| `migration.ai.max-input-tokens` | `8000` | Hard cap on tokens sent per request (system + user combined). `CompactPromptBuilder` trims content to stay under this limit. |
| `migration.ai.max-output-tokens` | `2000` | Hard cap on tokens generated per response. Increase if the AI truncates large diffs. |

---

## Console Settings (`migration.console.*`)

| Property | Default | Description |
|----------|---------|-------------|
| `migration.console.color-diff` | `true` | Show ANSI colour in unified diffs (green adds, red removes, cyan hunk headers). Set to `false` for terminals that don't support ANSI or for piped output. |
| `migration.console.auto-apply-threshold` | `0.95` | Reserved for future auto-apply logic. Not currently enforced. |

## Minimal Production `application.yml`

```yaml
migration:
  project-root: /path/to/active-pivot-project

  knowledge-dir: /opt/migration/knowledge
  jar-index-dir: /opt/migration/jar-index
  fixes-dir:     /opt/migration/fixes
  jars-dir:      /opt/migration/jars

  ai:
    provider: CLAUDE
    api-key:  ${ANTHROPIC_API_KEY}
    model:    claude-opus-4-8
    max-input-tokens:  12000
    max-output-tokens: 4000

  console:
    color-diff: true

spring:
  main:
    banner-mode: off
    web-application-type: none

logging:
  level:
    root: WARN
    com.bank.migration: INFO
```
